REQUIREMENTS            *copilot-status.nvim-copilot-status.nvim-requirements*

- Neovim 0.8+
- copilot.lua <https://github.com/zbirenbaum/copilot.lua>
- A patched font <https://www.nerdfonts.com> or a font like Nonicons <https://github.com/yamatsum/nonicons> (which I recommend)
- A place where you can show the status indicator, like lualine <https://github.com/nvim-lualine/lualine.nvim>

USAGE                          *copilot-status.nvim-copilot-status.nvim-usage*

`copilot-status` keeps track of two fields:

- **status** which is one of
    - "offline"
    - "loading"
    - "idle"
    - "warning"
    - "error"
- **message** which is only populated when an error occurs

You can get these values with `.status()` and manipulate them yourself, or use
the provided `.status_string()` function, which turns the status into an icon
and appends the optional "message" if it exists. You can
|copilot-status.nvim-configure| the icons yourself.


EXAMPLE WITH LUALINE ~

>lua
    require('lualine').setup {
      sections = {
        lualine_x = {
          function() return require("copilot_status").status_string() end,
          cnd = function() return require("copilot_status").enabled() end,
        }
      }
    }
<


CONFIGURATION          *copilot-status.nvim-copilot-status.nvim-configuration*

`copilot-status` has little settings to adjust at the moment. Here are the
defaults:

>lua
    require('copilot_status').setup({
      icons = {
        idle = " ",
        error = " ",
        offline = " ",
        warning = " ",
        loading = " ",
      },
      debug = false,
    })
<


  Make sure `.setup()` is run before `.string()` or `.status_string()`, otherwise
  the plugin will just ignore your configuration.
